# STEP URDF Maker 케이스별 설정 매뉴얼

이 문서는 STEP 형상을 URDF 링크와 관절로 재구성할 때 실제로 어떤 부모, 관절 종류, 축, 연동 방식을 선택해야 하는지 사례별로 정리합니다. 기능이 추가되거나 새로운 기구를 설정할 때 같은 형식으로 계속 확장합니다.

- 기준 버전: 로컬 개발 버전 · 프로젝트 형식 v3
- 마지막 갱신: 2026-07-22
- 표시 단위: 직선 이동 `mm`, 회전 `deg`, 바퀴 속도 `RPM`

## 가장 먼저 알아야 할 네 가지

### 링크는 함께 움직이는 형상 묶음이다

볼트로 단단히 결합되어 항상 같은 자세로 움직이는 STEP 형상은 하나의 링크로 묶습니다. 하나의 STEP 형상을 두 링크에 중복으로 배정하지 않습니다.

### 관절은 부모와 자식 사이의 동작 하나다

URDF 관절 하나는 기본적으로 한 가지 자유도만 표현합니다.

- `fixed`: 부모에 단단히 고정
- `prismatic`: 한 축을 따라 직선 이동
- `revolute`: 제한 범위 안에서 회전
- `continuous`: 제한 없이 계속 회전하는 바퀴 등에 사용

조향과 바퀴 굴림처럼 동작이 두 개라면 관절도 두 단계로 나눕니다.

### 부모와 구동원은 다르다

이 구분이 가장 중요합니다.

| 항목 | 의미 | 예시 |
|---|---|---|
| 부모 링크 | 자식의 기준 좌표계이며 부모의 이동을 강체로 상속 | 바퀴의 부모는 조향 너클 |
| mimic 구동 관절 | 자식 관절의 위치값을 계산하는 입력 | 너클 회전의 구동원은 핸들 |
| 속도 구동 레버 | `continuous` 관절의 회전속도를 계산하는 입력 | 바퀴 속도의 구동원은 전·후진 레버 |

부모를 지정했다고 자식 관절의 회전값까지 자동으로 바뀌지는 않습니다. 부모를 따라 위치는 이동하지만 자체 회전도 필요하면 mimic이나 속도 구동을 별도로 설정합니다.

### Base는 하나지만 로컬 고정부는 여러 개 만들 수 있다

하나의 URDF에는 부모가 없는 최상위 링크가 하나만 있어야 합니다. 보통 이름은 `base_link`입니다. 서로 떨어진 여러 장치를 구성하려면 두 번째 Base를 만들지 않고 `fixed` 자식 링크를 로컬 지지대로 사용합니다.

```text
base_link
├─ 핸들 지지대 (fixed)
│  └─ 핸들 (revolute)
└─ 모터 지지대 (fixed)
   └─ 회전체 (continuous)
```

## 공통 작업 순서

1. `Ctrl+S`로 현재 프로젝트를 저장합니다.
2. 3D 화면이나 형상 목록에서 실제로 함께 움직일 형상을 모두 선택합니다.
3. `+ 선택 형상으로 자식 링크`를 누릅니다.
4. 부모 링크와 동작 종류를 정합니다.
5. 관절 중심, 축 방향, 상태 0/1 값을 확인합니다.
6. 생성 창의 미리보기로 동작을 시험합니다.
7. 생성한 링크를 트리에서 다시 선택하고 오른쪽 패널에서 세부 값을 조정합니다.
8. 연동이 필요하면 위치 mimic 또는 주행 속도 구동 중 하나를 설정합니다.
9. `동작 설정 적용`을 누릅니다.
10. `▶ 자동`으로 전체 자동 동작을 확인하거나 `🎮 조작`으로 핸들과 주행 레버를 직접 시험한 뒤 다시 저장합니다.

현재 오른쪽 관절 편집기에서는 기존 링크의 부모 항목을 직접 바꾸지 않습니다. 부모를 잘못 선택했다면 해당 링크의 형상을 미할당으로 되돌린 뒤 올바른 부모에서 자식 링크를 다시 만드는 것이 안전합니다.

## 대표 기구 마법사로 시작하기

반복되는 단순 기구는 움직일 형상을 선택하고 `Ctrl+M`을 누르면 빠르게 시작할 수
있습니다. 문·뚜껑, 슬라이더·리프트, 연속 회전체, mimic 연동, 컨베이어 롤러 중
하나를 고른 뒤 부모 링크와 상태 0/1을 확인합니다. 완료하면 기존 3D 축 설정 창이
열리므로 축 후보, 관절 중심과 실제 움직임을 반드시 미리보기로 검사합니다.

마법사는 일반 편집을 대체하지 않습니다. 만든 뒤 오른쪽 관절 편집기에서 동작 종류,
범위, effort, velocity, damping, friction, mimic 또는 속도 입력을 다시 수정할 수
있습니다. 이 물리값은 표준 URDF로 저장되지만 제조사 사양을 모르는 상태에서 제공되는
값은 안전한 편집 시작값일 뿐이므로 실제 동역학 시뮬레이션 전에는 질량·관성 및
구동기 사양과 함께 보정해야 합니다.

컨베이어 마법사는 롤러의 `continuous` 관절과 속도 의미를 만듭니다. 박스가 벨트
위에서 실제로 이동하는 동작은 URDF 관절만으로 정의되지 않으므로 Gazebo 등의 접촉
마찰 또는 컨베이어 플러그인을 추가해야 합니다. 마법사 의미와 상태 이름은 내보낸
패키지의 `config/mechanisms.json`에 남아 후속 시뮬레이션 설정에서 사용할 수 있습니다.

## 축 추천과 직접 조정

새 자식 링크를 만들 때 축 후보는 다음 순서로 계산됩니다.

1. 부모 형상과 자식 형상에 동일 중심선의 원통면이 있으면 그 CAD 중심축을 우선합니다.
2. 공통 중심선이 없으면 자식 형상의 BBox/PCA 방향을 사용합니다.
   - A: 가장 긴 가로 장축
   - B: 두 번째 세로축
   - C: 가장 얇은 두께축
3. 자동 후보가 맞지 않으면 `3D에서 축 위치·방향 직접 조정`을 사용합니다.

회전 중심은 단순히 형상 BBox 중심이 아니라 실제 베어링, 핀, 원통 결합부의 중심선이어야 합니다. 노란 조절선의 양 끝은 방향, 가운데는 중심 위치를 조절합니다.

## 상태 0/1 사용법

`상태 0/1 실제값`은 관절의 양 끝과 연동 계산 기준입니다.

- `prismatic`: mm 단위, 예: `-45 mm → +45 mm`
- `revolute`: deg 단위, 예: `-30° → +30°`
- `continuous`: 위치 한계 없이 계속 회전하므로 상태 범위는 일반 동작 시험용 한 바퀴 범위입니다.

mimic 자동 계산을 사용하면 구동 관절의 상태 0/1이 종속 관절의 상태 0/1에 선형으로 대응됩니다.

## CASE-01. 움직이지 않는 차체와 고정부

### 목표

차체, 프레임, 커버처럼 움직이지 않는 형상을 기준 링크로 구성합니다.

### 권장 구성

```text
base_link
├─ 고정 프레임 A (fixed, 선택 사항)
└─ 고정 프레임 B (fixed, 선택 사항)
```

### 설정

- 항상 같은 강체라면 형상을 모두 `base_link`에 포함합니다.
- 구역별로 부모를 명확하게 나누고 싶다면 해당 형상을 별도 링크로 만들고 관절을 `fixed`로 설정합니다.
- 빈 fixed 링크를 근처에 놓는 것만으로 주변 형상이 자동 수집되지는 않습니다. 축 판단에 사용할 지지대·하우징 형상을 그 링크에 실제로 배정해야 합니다.

### 확인

Play나 다른 관절 슬라이더를 움직여도 고정부 자체 자세가 변하지 않아야 합니다.

## CASE-02. 단일 직선 이동 장치

### 목표

실린더 로드, 슬라이더, 승강축 등을 한 방향으로 이동시킵니다.

### 권장 구성

```text
고정 지지대
└─ 이동 뭉치 (prismatic)
```

### 설정

- 부모: 실제 가이드나 하우징 링크
- 종류: `prismatic`
- 중심: 이동 뭉치의 기준점 또는 결합 중심
- 축: 실제 가이드 방향
- 상태 0/1: 실제 최소·최대 스트로크

### 흔한 오류

- 이동 방향이 반대라면 `방향 반전` 또는 반대 부호 축을 사용합니다.
- 화살표 위치는 선택 링크 전체 BBox 중심에 표시되지만 실제 이동축 계산은 설정한 관절축을 사용합니다.

## CASE-03. 단일 제한 회전 장치

### 목표

핸들, 힌지, 레버처럼 일정 각도 범위에서 회전시킵니다.

### 권장 구성

```text
고정 지지대
└─ 회전 뭉치 (revolute)
```

### 설정

- 부모: 회전 베어링이나 고정 하우징 링크
- 종류: `revolute`
- 중심: 실제 핀 또는 원통 결합 중심선 위의 점
- 축: 실제 중심축 방향
- 상태 0/1: 예를 들어 `-180° → +180°` 또는 `-45° → +45°`

### 확인

주황색 회전축이 실제 결합 중심선을 통과해야 합니다. 슬라이더 양 끝에서 형상이 원하는 각도까지 회전하는지 확인합니다.

## CASE-04. 부모와 완전히 붙어서 따라가는 부품

### 목표

센서, 커버, 고정 브래킷 또는 조향 너클에 붙은 부품이 부모와 같은 움직임을 하게 합니다.

### 권장 구성

```text
움직이는 부모 링크
└─ 고정 부품 (fixed)
```

### 결과

자식은 부모의 직선 이동과 회전을 모두 그대로 따라갑니다. 자체 회전이나 이동은 없습니다.

## CASE-05. 핸들 회전으로 중앙 축을 좌우 이동

### 목표

핸들을 돌리면 별도의 조향축 또는 랙 링크가 좌우로 이동하게 합니다.

### 권장 트리

```text
base_link
├─ 핸들 (revolute)
└─ 중앙 조향축 (prismatic)
```

두 링크가 트리상 형제여도 mimic으로 동작을 연결할 수 있습니다.

### 설정

1. 핸들 관절의 상태 0/1을 예: `-180° → +180°`로 설정합니다.
2. 중앙 축 관절의 상태 0/1을 예: `-45 mm → +45 mm`로 설정합니다.
3. 중앙 축 관절을 선택합니다.
4. `다른 관절과 연동(mimic)`을 켭니다.
5. 구동 관절로 핸들 관절을 선택합니다.
6. `상태 0↔1에서 배율 자동 계산`을 유지합니다.
7. 방향이 반대면 `반대 방향으로 연동`을 켭니다.

### 결과

```text
핸들 -180° → 중앙 축 -45 mm
핸들    0° → 중앙 축   0 mm
핸들 +180° → 중앙 축 +45 mm
```

## CASE-06. 이동 축을 따라가면서 너클도 회전

### 목표

중앙 축이 이동하면 그 자식 너클이 축과 함께 이동하면서 동시에 자체 회전합니다.

### 단순화한 권장 트리

```text
base_link
├─ 핸들 (revolute)
└─ 중앙 축 (prismatic, 핸들 mimic)
   └─ 왼쪽 너클 (revolute, 핸들 mimic)
```

### 핵심 설정

- 왼쪽 너클의 부모 링크: 중앙 축
- 왼쪽 너클의 mimic 구동 관절: 핸들 관절

부모가 중앙 축이므로 좌우 이동을 상속하고, mimic 구동원이 핸들이므로 자체 회전값도 바뀝니다.

대신 중앙 축 이동량을 기준으로 회전시키려면 너클의 구동 관절로 중앙 축 관절을 선택할 수도 있습니다. 현재는 둘 다 선형 근사입니다.

### 예시

- 핸들: `-180° → +180°`
- 중앙 축: `-45 mm → +45 mm`
- 너클: `-30° → +30°`

## CASE-07. 실제 연결봉 구조를 단순화

### 상황

조향 로드가 너클의 팔을 당기면서 회전시키는 구조는 실제로 폐루프 또는 다절 링크가 될 수 있습니다.

### 현재 권장 방식

연결봉의 모든 접촉을 URDF 부모·자식으로 강제로 표현하지 않고 최종 결과 운동을 직접 설정합니다.

```text
핸들 → 중앙 이동축 mimic → 너클 회전 mimic
```

### 한계

현재 mimic은 선형입니다. 실제 애커먼 조향처럼 좌우 각도가 비선형으로 달라지는 관계는 상태 0/1 값으로 근사합니다. 실제 기하 관계가 필요하면 별도 기구 구속조건 솔버가 필요합니다.

## CASE-08. 바퀴를 조향 링크에 고정해 방향만 바꾸기

### 목표

바퀴가 굴러가지는 않지만 너클을 따라 조향 방향으로 움직이게 합니다.

### 권장 구성

```text
왼쪽 너클 (revolute)
└─ 왼쪽 바퀴 (fixed)
```

### 결과

바퀴는 너클의 이동과 조향 회전을 모두 따라갑니다. 자체 굴림은 없습니다.

## CASE-09. 바퀴 굴림만 추가

### 목표

기존 STEP 바퀴를 자체 가로축을 중심으로 계속 회전시킵니다.

### 권장 구성

```text
고정 차축 또는 조향 너클
└─ 바퀴 (continuous)
```

### 설정

1. 타이어와 함께 회전할 휠 형상을 모두 선택합니다.
2. 부모는 실제로 바퀴를 지지하는 차축 또는 너클 링크를 선택합니다.
3. 종류를 `continuous`로 설정합니다.
4. 중심을 바퀴 중심으로 설정합니다.
5. 축은 바퀴 가로 중심축으로 설정합니다.

바퀴 형상이 아직 `base_link`에 들어 있다면 새 바퀴 링크로 옮겨야 합니다. STEP 파일 자체를 수정할 필요는 없습니다.

## CASE-10. 조향과 굴림을 모두 담당하는 바퀴

### 목표

바퀴 전체가 좌우로 조향하면서 동시에 자체 축으로 굴러가게 합니다.

### 필수 2단 구성

```text
고정 차축
└─ 조향 너클 (revolute, 핸들 mimic)
   └─ 바퀴 (continuous, 속도 구동)
```

### 형상 배정

- 조향 너클 링크: 너클, 조향 암, 고정 허브 형상
- 바퀴 링크: 타이어와 함께 실제로 굴러가는 휠 형상

### 주의

바퀴 하나에 조향과 굴림을 중복 설정하지 않습니다. 부모 너클의 회전과 자식 바퀴의 연속 회전이 합성되어 두 동작이 동시에 보입니다.

## CASE-11. 전·후진 레버 위치로 바퀴 속도 제어

### 목표

레버를 앞으로 보낼수록 바퀴가 빠르게 정회전하고, 중앙에서는 정지하며, 뒤로 보낼수록 빠르게 역회전하게 합니다.

### 권장 트리

```text
base_link
├─ 전·후진 레버 (revolute 또는 prismatic)
├─ 왼쪽 구동 바퀴 (continuous)
└─ 오른쪽 구동 바퀴 (continuous)
```

레버와 바퀴는 부모·자식일 필요가 없습니다. 속도 구동 설정이 값을 연결합니다.

### 레버 설정

- 종류: `revolute` 또는 `prismatic`
- 예시 범위: `-20° → +20°`
- 상태 0: 최대 후진
- 정확한 중앙: 중립
- 상태 1: 최대 전진

현재는 F/N/R 버튼으로 딱 세 위치에 스냅하는 방식이 아니라, 레버 위치에 따라 방향과 속도가 연속적으로 바뀌는 방식입니다. 중앙 주변은 `중립 범위`로 정지 구간을 만들 수 있습니다.

### 바퀴 설정

1. 바퀴 관절을 `continuous`로 설정합니다.
2. 바퀴 관절의 오른쪽 `주행 구동`을 엽니다.
3. `이 연속 회전 관절을 레버로 구동`을 켭니다.
4. `전·후진 레버`에서 레버 관절을 선택합니다.
5. 상태 0/1의 최대 RPM을 입력합니다.
6. 중앙 정지를 위한 중립 범위를 입력합니다. 기본값은 3%입니다.
7. 바퀴 회전 방향이 반대면 `바퀴 회전 방향 반전`을 켭니다.
8. `동작 설정 적용`을 누릅니다.

### 시험

1. 3D 화면 오른쪽 아래의 `🎮 조작`을 누릅니다.
2. 설정용 좌우 패널이 숨겨지고 3D 화면에 `연동 입력`과 `주행 레버`만 나타나는지 확인합니다.
3. 주행 레버 슬라이더를 움직입니다.
4. 중앙에서 정지하고 양쪽에서 반대 방향으로 회전하는지 확인합니다.
5. `조작 정지`를 누르면 재생 전 관절 위치와 설정 패널이 복원됩니다.

자동/조작 Play 모두 카메라를 자동으로 회전하지 않으며 사용자가 정한 시점을 유지합니다. 필요하면 재생 중에도 3D 뷰를 직접 회전·확대해 확인합니다.

### 자동 Play와 조작 Play의 차이

| 모드 | 용도 | 화면에 나타나는 입력 |
|---|---|---|
| `▶ 자동` | 전체 동작 데모 | 독립 관절이 자동 왕복 |
| `🎮 조작` | 실제 운전자 조작 시험 | mimic 원본 핸들과 속도 구동 레버만 표시 |

조작 Play의 패널은 링크 이름으로 임의 판단하지 않습니다. 다른 관절이 mimic으로 참조하는 원본 관절과 바퀴 주행 구동이 참조하는 레버 관절을 프로젝트 설정에서 자동으로 찾습니다.

### 계산 개념

```text
상태 0       → -최대 RPM
중앙         → 0 RPM
상태 1       → +최대 RPM
중간 위치    → 위치에 비례한 RPM
```

## CASE-12. 좌우 구동 바퀴를 같은 레버에 연결

### 설정

왼쪽과 오른쪽 바퀴를 각각 선택해 CASE-11의 주행 구동 설정을 반복하고 같은 레버 관절을 선택합니다.

두 바퀴의 관절축을 같은 월드 방향으로 만들었다면 같은 부호로 회전합니다. 모델링 과정에서 축 부호가 서로 반대라면 한쪽 바퀴에만 `바퀴 회전 방향 반전`을 켭니다.

## CASE-13. 별도 가속 페달과 F/N/R 선택기

### 목표 구조

```text
바퀴 속도 = 방향 선택값 × 가속 페달값 × 최대 속도
```

- 전진: `+1`
- 중립: `0`
- 후진: `-1`
- 가속 페달: `0.0 → 1.0`

### 현재 지원 상태

현재 앱에 구현된 주행 구동은 레버 하나의 위치로 방향과 속도를 함께 제어합니다. 별도의 3단 F/N/R 선택기와 가속 페달을 두 입력으로 조합하는 기능은 아직 구현되지 않았습니다. 이 경우 위치 mimic으로 대체하지 말고 향후 전용 주행 제어 기능을 사용해야 합니다.

## CASE-14. 여러 독립 회전체를 한 로봇에 추가

### 권장 구성

```text
base_link
├─ 장치 A 지지대 (fixed)
│  └─ 장치 A 회전체 (revolute 또는 continuous)
└─ 장치 B 지지대 (fixed)
   └─ 장치 B 회전체 (revolute 또는 continuous)
```

회전체를 만들 때 가까운 고정 지지대 링크를 부모로 선택하면, 축 추천은 그 부모 형상과 선택한 자식 형상의 공통 원통 중심선을 우선 사용합니다. 단순한 공간적 근접성만으로 배정되지 않습니다.

## 문제 해결표

| 증상 | 주된 원인 | 해결 방법 |
|---|---|---|
| 자식이 부모를 따라 이동하지만 회전하지 않음 | 부모 관계만 있고 자식 관절값을 구동하지 않음 | 자식 `revolute` 관절에 mimic 구동 관절 지정 |
| 바퀴가 너클 회전을 따라가지 않음 | 바퀴 부모가 base 또는 다른 차축임 | 바퀴 부모를 너클로 구성 |
| 기존 링크의 부모를 바꾸고 싶음 | 오른쪽 부모 항목은 읽기 전용임 | 형상을 미할당으로 되돌리고 올바른 부모 아래 자식 링크로 다시 생성 |
| 바퀴가 조향은 하지만 굴러가지 않음 | 바퀴 관절이 `fixed`임 | 바퀴 관절을 `continuous`로 변경하고 축 설정 |
| 조작 Play에서 속도 구동 바퀴가 돌지 않음 | 레버가 중앙이거나 설정 적용 전임 | 주행 레버를 중앙 밖으로 옮기고 바퀴 설정 적용 확인 |
| 조작 Play에 핸들/레버가 나타나지 않음 | 어떤 관절도 해당 입력을 참조하지 않음 | 조향 mimic 또는 바퀴 주행 구동에서 원본 관절을 먼저 지정 |
| 바퀴가 반대로 회전함 | 바퀴 축 부호가 반대임 | `바퀴 회전 방향 반전` 사용 |
| mimic 종속 관절 슬라이더가 비활성화됨 | 종속 관절은 직접 움직이지 않도록 설계됨 | mimic 구동 관절의 슬라이더 사용 |
| 주행 구동을 켤 수 없음 | 바퀴 관절이 `continuous`가 아님 | 동작 종류를 `continuous`로 변경 |
| 레버가 구동원 목록에 없음 | 레버가 fixed/continuous이거나 설정 적용 전임 | 레버를 `revolute`/`prismatic`으로 만들고 적용 |
| 원하는 가로축 후보가 없음 | 자식 원통축 또는 잘못된 형상 선택 | 창을 다시 열어 BBox A/B/C 확인, 필요하면 3D 직접 조정 |
| 회전축 위치가 바퀴 중심에 잡힘 | 굴림축과 조향축을 혼동 | 조향은 너클 핀 중심선, 굴림은 바퀴 가로 중심선 사용 |
| 루트 링크가 여러 개라는 오류 | 서로 연결되지 않은 별도 트리 생성 | 하나의 base 아래 fixed 지지대로 연결 |
| 같은 형상이 이상하게 중복 이동함 | 형상을 여러 기능 링크처럼 구성하려 함 | STEP 형상 하나는 한 링크에만 배정 |

## 저장과 내보내기

- 링크·관절·mimic·주행 속도 구동 설정은 `.urdfmaker.json` 프로젝트에 저장됩니다.
- 프로젝트 형식 v4는 이전 v1/v2/v3 프로젝트를 읽을 수 있습니다.
- 마법사에서 만든 기구 의미와 상태 이름은 `config/mechanisms.json`으로도 내보냅니다.
- URDF 내보내기에서 BBox 기반 근사 관성을 선택할 수 있지만 실제 CAD 질량 특성으로 교체해야 합니다.
- 표준 URDF의 `<mimic>`은 위치 연동만 표현하므로 mimic 설정은 URDF에 내보낼 수 있습니다.
- 전·후진 레버의 조건부 속도 구동은 순수 URDF에 표현할 수 없습니다. 실제 ROS 2 또는 시뮬레이터에서는 별도 컨트롤러가 필요합니다.
- 미리보기 관절 위치와 자동/조작 Play 중 누적 회전은 실제 초기 관절 상태로 URDF에 저장되지 않습니다.
